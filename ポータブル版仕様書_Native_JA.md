# C-Box TTS Native Japanese Edition 仕様書 (ポータブル版)

本ソフトウェアは、Python 環境や複雑な外部依存を一切必要とせず、Windows 上で完全ローカルかつ超軽量・高速に動作する .NET/C# ネイティブ音声合成アプリケーション「C-Box TTS Japanese Edition」の日本語用ポータブル配信パッケージです。

DirectML を採用し、NVIDIA、AMD、Intel 等の各種 GPU によるハードウェアアクセラレーションをネイティブにサポートしています。

---

## 1. パッケージフォルダ構成

`Release_Portable_JA/` 内は以下のように構成されています。

```text
Release_Portable_JA/
├── CBoxTTS.Native.JA.exe    # 日本語版メイン実行ファイル (GUI / CLI 両対応)
├── CBoxTTS.Native.dll       # コアロジックライブラリ
├── onnxruntime.dll          # ONNX Runtime 推論エンジン (C++ ネイティブ)
├── DirectML.dll             # DirectML テンソル演算ライブラリ (GPU加速用)
├── MeCab.DotNet.dll         # C#用 MeCab バインディング
├── dic/                     # 形態素解析 (MeCab IPADIC) 辞書フォルダ
│   ├── sys.dic              # システム辞書
│   └── (その他の関連辞書ファイル)
├── models/                  # 音声合成 ONNX AI モデルフォルダ
│   ├── default_voice.wav    # デフォルトの参照音声（クローニング用）
│   ├── turbo/               # 日本語専用・超高速モデル (ONNX)
│   └── multilingual/        # 多言語モデル (ONNX)
└── debug.log                # [自動作成] 実行トレース・デバッグログ
```

## 2. システムの特徴・独自改善

### ① 日本語専用「Turbo」および「Multilingual」モデルの搭載
日本語の読み上げに特化し、低遅延で自然な発話が可能な「Turbo」モデルと、高品質で声質クローニングに対応した「Multilingual」モデルを同梱しています。

### ② 濁音・半濁音の分離バグ修正 (Unicode NFC 正規化)
C# の標準トークナイザー処理において濁音（例：「が」）や半濁音（例：「ぱ」）が清音と濁点・半濁点に分離され、モデルのトークン埋め込み層でインデックス範囲外エラーを引き起こす不具合を解消しました。Unicode 正規化形式に **NFC (FormC)** を導入したことにより、文字の結合状態を完璧に維持し、極めて高品質で安定したエンコードを保証します。

### ③ 境界インデックス超過セーフガード (安全装置)
トークナイザーが文字IDを引き当てる際、ONNXモデルの埋め込み層上限インデックス（**`2453`**）を超える未知文字や全角記号（例：全角の「？」）が検出された場合、クラッシュを防止するために自動的に未知語トークン `1 (UNK)` にクリップ・マッピングする安全ガード回路を搭載しています。

### ④ 音声の頭切れ・ぶつ切り対策 (無音パディング)
再生デバイスの起動遅延やフェードインによる冒頭音声の欠落（頭切れ）を防ぐため、合成音声の冒頭に **`0.15秒`**、末尾に **`0.10秒`** の無音パディングを自動的に追加する処理を実装しました。これにより再生およびWAV保存時の安定性が大幅に向上しています。

### ⑤ 詳細パラメーター「安定性 (Stability)」の追加と自動最適化
トークン生成時のサンプリング確率のばらつき（Temperature）を制御する「安定性 (Stability)」スライダーを GUI に追加しました。また、モデルの種類に応じて最適なデフォルト値（Turbo: `0.6`、Multilingual: `0.7`）を自動設定する機能を搭載し、不要な吃りやノイズを抑えて明瞭な滑舌を実現しました。

### ⑥ CFG（Classifier-Free Guidance）の実装による流暢性の劇的向上
Python リファレンス実装（Chatterbox）に準拠した **CFG（Classifier-Free Guidance）** を移植しました。推論ループの各ステップで「条件付きテキスト」と「無条件（空テキスト）」の2つの推論パスを同時に処理し、得られた logits（対数尤度）を補間することで、音声の品質とスラスラとした流暢さを飛躍的に高めることができます。GUI に「CFG/ペース」スライダー（デフォルト値: `0.5`）が追加され、ユーザー自身で調整可能です。

### ⑦ ランダムシード非固定化による表現の多様化
音声のサンプリング処理における乱数生成シード（従来は `42` に固定）の固定化を撤廃しました。同じテキスト・同じ話者特徴を入力した場合でも、実行ごとにイントネーションや発話ピッチに自然なバリエーションを持たせ、ロボット的な同一出力を防いでいます。

### ⑧ 参照音声（ボイスプロンプト）の自動前処理
音声エンコーダに渡す前に、参照音声の先頭・末尾の無音区間を自動トリムし、10秒以内に制限する前処理を実装しました。これにより、録音環境によるノイズや長すぎるサンプルの影響を排除し、話者特徴の抽出精度が向上します。

### ⑨ セキュリティ・堅牢性の強化
包括的なセキュリティ監査に基づき、HTTPダウンロード時の不完全ファイル保護、スレッドセーフなログ機構、入力テキスト長制限（10,000文字）、リソースリーク防止など、12項目の脆弱性を修正しました。

## 3. 使用方法

### GUI (ウィンドウ起動)
`CBoxTTS.Native.JA.exe` をダブルクリックして起動します。
美麗なモダン WPF UI が表示され、テキストの入力、パラメータ（話速、抑揚、感情等）の調整、およびリアルタイムでの高品質な音声合成・ファイル出力・再生が簡単に行えます。

### CLI テストハーネス (コマンドライン起動)
コマンドプロンプトまたは PowerShell から以下の引数を指定して起動することで、GUI を表示させずに一気通貫の合成合成テストを実行し、動作確認用の WAV ファイルを出力できます。

```powershell
.\CBoxTTS.Native.JA.exe --test
```

*   **動作ログ**: `test_harness.log` に処理ステップ（形態素解析 -> トークナイズ -> 推論 -> 音声出力）の全記録が出力されます。
*   **出力ファイル**: 合成された日本語音声が `test_harness_japanese_out.wav` などとして保存されます。
