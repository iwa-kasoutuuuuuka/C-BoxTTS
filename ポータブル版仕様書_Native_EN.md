# C-Box TTS Native English Edition 仕様書 (ポータブル版)

本ソフトウェアは、Python 環境や複雑な外部依存を一切必要とせず、Windows 上で完全ローカルかつ超軽量・高速に動作する .NET/C# ネイティブ音声合成アプリケーション「C-Box TTS English Edition」の英語用ポータブル配信パッケージです。

DirectML を採用し、NVIDIA、AMD、Intel 等の各種 GPU によるハードウェアアクセラレーションをネイティブにサポートしています。

---

## 1. パッケージフォルダ構成

`Release_Portable_EN/` 内は以下のように構成されています。

```text
Release_Portable_EN/
├── CBoxTTS.Native.EN.exe    # 英語版メイン実行ファイル (GUI / CLI 両対応)
├── CBoxTTS.Native.dll       # コアロジックライブラリ
├── onnxruntime.dll          # ONNX Runtime 推論エンジン (C++ ネイティブ)
├── DirectML.dll             # DirectML テンソル演算ライブラリ (GPU加速用)
├── models/                  # 音声合成 ONNX AI モデルフォルダ
│   ├── default_voice.wav    # デフォルトの参照音声（クローニング用）
│   ├── english/             # 英語専用・高品質モデル (ONNX)
│   └── multilingual/        # 多言語モデル (ONNX)
└── debug.log                # [自動作成] 実行トレース・デバッグログ
```

> [!NOTE]
> **MeCabおよび辞書フォルダの排除による超軽量化**:
> 英語専用パッケージでは、日本語の形態素解析（MeCab）関連ファイル（`MeCab.DotNet.dll` および `dic/` 辞書ディレクトリ）を完全に排除したことにより、パッケージサイズとメモリ使用量を劇的にスリム化しました。

## 2. システムの特徴・独自改善

### ① 英語完全対応のUIローカライズ
英語版（`Release_EN` ビルド）では、アプリケーションのウィンドウタイトル、ラベル、ボタン、および詳細設定のテキストが完全に英語でローカライズ表示されます。

### ② 英文テキスト正規化 (English Normalizer) の大幅なデバッグと強化
英文の読み上げ時において、以下の表記が分断されず高品質に発音されるよう、テキスト正規化エンジンを包括的に強化しました。
*   **略語展開**: `Mr.` → `mister`, `Dr.` → `doctor`, `etc.` → `et cetera`、月名略語（`Jan.`〜`Dec.`）等の自動展開。
*   **序数変換**: `1st` → `first`, `22nd` → `twenty-second` 等の序数を自然な英語読みに変換。
*   **時刻変換**: `3:00` → `three o'clock`, `3:30 pm` → `three thirty pm` 等の時刻表記を展開。
*   **年号変換**: `2024` → `twenty twenty-four`, `1999` → `nineteen ninety-nine` 等の年号を自然な読みに変換。
*   **分数変換**: `1/2` → `one half`, `3/4` → `three quarters` 等の分数を展開。
*   **英語短縮形の保護**: `don't` や `it's` などのアポストロフィを含む短縮形が BPE トークナイザーに渡る前段で分解されず適切に保持されるよう、英語テキスト時の NFD 正規化をスキップさせ保護を強化。
*   **PC・FA業界専門用語（User Dictionary）への対応**: IT/PC業界用語（`CPU`, `GPU`, `SSD` 等）やFA（ファクトリーオートメーション）業界用語（`PLC`, `OMRON`, `SCADA` 等）の自然な発音を記述した英語ユーザー辞書（`user_dict_en.txt`）を標準搭載。
*   **一般英単語の誤スペルアウト防止**: 2〜6文字の大文字頭字語のスペルアウト処理において、全大文字文の中の一般的な英単語（`IT`, `IS`, `DAY` 等）が誤ってスペルアウトされるのを防ぐ除外保護リスト（`KnownCommonWords`）を追加。
*   **頭字語**: FBI, NASA, SCADA, BIOS 等の既知の大文字頭字語を不自然に展開・破壊せず、そのままの単語として滑らかに発音可能に保持。
*   **カンマ区切り数値**: `12,345` → `twelve thousand three hundred and forty-five` へ自動変換。
*   **小数表記**: `12.34` → `twelve point three four` のように小数点以下を各桁読みで展開。
*   **通貨表記**: `$1.50` → `one dollar and fifty cents`, `£100` → `one hundred pounds` 等、ドル・ポンドに対応。
*   **記号置換**: `%` → `percent`, `@` → `at`, `#` → `number`, `+` → `plus`, `=` → `equals`。
*   **URL/email除去**: 発音不能なURLやメールアドレスを自動的に除去。

### ③ 英語専用モデルにおける発音不全の解消
英語専用モデルのロード時に、言語ID（`1`）の不一致によりテキスト正規化およびトークナイザーの小文字化・記号正規化（BPE辞書用処理）が誤ってスキップされていた問題を根本修正しました。これにより、英語専用モデル使用時でも常に最高品質の発音性能を発揮します。

### ④ 音声の頭切れ対策 (無音パディング)
再生環境で生じる開始時の音声欠損（頭切れ）を吸収するため、合成されたすべての音声波形の冒頭に **`0.15秒`**、末尾に **`0.10秒`** の無音パディングを自動追加する処理を実装しました。

### ⑤ 詳細パラメーター「安定性 (Stability)」の追加と最適値の適用
自己回帰推論時の安定性パラメータ（Temperature）を制御するスライダーを GUI に追加しました。英語専用モデルは語彙サイズが小さく揺らぎの影響を受けやすいため、デフォルト値を **`0.5`** に設定することで、不要な吃り、かすれ、ノイズを完全に排除し、滑らかでクリアな滑舌を実現しました。（多言語モデルでの英語合成は `0.7`）

### ⑥ CFG（Classifier-Free Guidance）の実装による流暢性の劇的向上
Python のリファレンス実装（Chatterbox）と同様の **CFG（Classifier-Free Guidance）** を C# ネイティブ推論エンジンへ追加移植しました。推論時に「条件付き（テキストあり）」と「無条件（テキストなし）」の2つのパスを同時に実行し、その結果を補間（補間重み: デフォルト `0.5`）することにより、英語の長文テキストの読み上げ時において、スラスラとした極めて流暢で自然な発音出力を得ることが可能になりました。GUI に「CFG/ペース」スライダーが追加され、手動で調整できます。

### ⑦ ランダムシード非固定化による表現の多様化
自己回帰サンプリングに用いられる乱数のシード固定値（従来は `42` で固定）を撤廃し、動的シードに変更しました。これにより、同じテキストの読み上げにおいても、ピッチやイントネーションなどに実質的に自然な多様性と人間味を持たせることが可能になりました。

### ⑧ 参照音声（ボイスプロンプト）の自動前処理
音声エンコーダに渡す前に、参照音声の先頭・末尾の無音区間を自動トリムし、10秒以内に制限する前処理を実装しました。これにより、録音環境によるノイズや長すぎるサンプルの影響を排除し、話者特徴の抽出精度が向上します。

### ⑨ セキュリティ・堅牢性の強化
包括的なセキュリティ監査に基づき、以下の強化を実施しました。
*   HTTPダウンロード時の一時ファイル保護（不完全なモデルファイルによるクラッシュを防止）
*   スレッドセーフなログ機構（推論スレッドとUIスレッドの競合を防止）
*   入力テキスト長制限（10,000文字、メモリ圧迫を防止）
*   リソースリーク防止（Dispose後のnullリセット、AudioEngineの確実な解放）
*   OnnxGqaPatcherのパッチ済みマーカーによる再実行防止

## 3. 使用方法

### GUI (ウィンドウ起動)
`CBoxTTS.Native.EN.exe` をダブルクリックして起動します。
英語表記の WPF UI が表示され、テキストの入力、パラメータ（話速、抑揚、感情等）の調整、およびリアルタイムでの高品質な音声合成・ファイル出力・再生が簡単に行えます。

### CLI テストハーネス (コマンドライン起動)
コマンドプロンプトまたは PowerShell から以下の引数を指定して起動することで、GUI を表示させずに一気通貫の合成合成テストを実行し、動作確認用の WAV ファイルを出力できます。

```powershell
.\CBoxTTS.Native.EN.exe --test
```

*   **動作ログ**: `test_harness.log` に処理ステップ（トークナイズ -> 推論 -> 音声出力）の全記録が出力されます。
*   **出力ファイル**: 合成された英語音声が `test_harness_english_exclusive_out.wav` などとして保存されます。
