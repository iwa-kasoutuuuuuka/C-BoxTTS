# C-Box TTS Native (Japanese & English Edition)

<p align="center">
  <img src="CBoxTTS.Native/assets/icon.png" alt="C-Box TTS Logo" width="128" height="128" />
</p>

完全ローカルかつ超軽量・高速に動作する .NET/C# ネイティブ（WPF）音声合成（TTS）アプリケーションです。  
Python 環境や PyTorch などの巨大な外部依存を一切必要とせず、Windows 上でスタンドアロンかつ即座に動作します。

---

## 🌟 主な特徴

- **完全ローカル・オフライン動作**: 初回起動時のモデル取得後は一切通信を行わず、プライバシーを完全に保護。
- **CUDA / DirectML ネイティブサポート**: GPU（NVIDIA CUDA / DirectML）によるハードウェア加速に対応。NVIDIA RTX 50 シリーズ等の最新 GPU 環境でも安定して高速推論可能。
- **英語モデルの発音・アクセント（訛り）の根本解決 & US Native 化**:
  - **トークン処理バグの修正**: 英語専用モデル使用時に文頭へ不要な言語トークン `1` (UNK) が自動挿入されていた問題を特定・修正。モデル内部の位置埋め込み（`position_ids`）の1マスズレを解消。
  - **標準アメリカ英語 (General American Accent / US Native) 参照音声の同梱**: クリアなアメリカ英語ネイティブ参照音声 (`default_voice_en.wav`, `default_voice_en_female.wav`, `default_voice_en_male.wav`) を標準同梱。参照音声の転写によるカタカナ風の訛りを完全に排除し、Native Speaker 同等の綺麗な英語発音を実現。
  - **パラメータの自動最適化**: 英語モデル選択時、Exaggeration (抑揚/強調): `0.30`、Temperature: `0.50`、CFG Weight: `0.40` へパラメータを自動調整し、癖のないクリアな発音にアラインメント。
- **話速調整における「声の二重重なり・ダブり」の完全排除 (WSOLA 刷新)**:
  - タイムドメイン **WSOLA（Waveform Similarity Overlap-Add）** アルゴリズムを全面再設計。話速を遅く（ゆっくり）設定した際に発生していた声のエコー・ディレイ・重なり現象を完全に克服し、どの話速でも歪みのない極めてクリアで自然な音質を実現。
- **テキスト正規化 (English Normalizer) & 英語ユーザー辞書**:
  - 英語読み上げ時において、略語展開（`Mr.`, `Dr.`, 月名略語等）、序数（`1st` → `first`）、時刻（`3:00` → `three o'clock`）、年号（`2024` → `twenty twenty-four`）、分数（`1/2` → `one half`）、頭字語（`FBI`, `NASA` 等の不自然なスペルアウト破壊を防ぐ保護）、通貨・セント表記（`$1.50` → `one dollar and fifty cents`、`£100` → `one hundred pounds`）、小数（`3.14` → `three point one four`のpoint/各桁展開）、記号（`%` → `percent`、`@` → `at` 等）、URL/email除去 を含む包括的なテキスト展開で自然な発音を実現。
  - 英語専用ユーザー辞書（`user_dict_en.txt`）を標準同梱。単語境界（`\b`）を厳密に考慮した正規表現置換により、PC/FA業界用語（CPU, GPU, PLC, SCADA等）や固有名詞のカスタマイズ読み仮名登録が可能。
- **英語読み上げ流暢性・結合処理の大幅な向上**:
  - 英語一括合成の文字数閾値を 150 文字から **350 文字** へ大幅に引き上げ。自己回帰推論の最大トークン数を 400 から **760** へ拡張。
  - 短いセンテンスが細切れに合成されて発音が崩れるのを防ぐ、英語専用の「短文自動結合ロジック (`MergeShortSentencesForEnglish`)」を搭載。
  - 50ms（1200サンプル）の「クロスフェード結合 (`CrossfadeJoinChunks`)」を実装。クリックノイズや不自然な間（ギャップ）を完全に排除したスラスラとした流暢な英語音声を生成（※日本語版は句読点での感情表現や自然な間を活かす無音ギャップ結合を維持）。
- **テキスト処理のセーフガードとトークナイザーの動的制限**:
  - Unicode NFC (FormC) 正規化の導入により、濁音・半濁音の文字分離バグを根本解決。
  - 語彙辞書の読み込み時に、ロードされたモデルの最大語彙インデックス（`MaxValidTokenId`）を動的に検出するようにトークナイザーを改善。日本語Turboモデルだけでなく英語（語彙数6561）や多言語モデルにおいても、有効な文字トークンが誤って `UNK (1)` に安全マッピングされることなく本来の発音性能を100%発揮可能。
- **音声の頭切れ・ぶつ切り対策 (無音パディング & フェード適用)**:
  - 合成音声の冒頭（0.15秒）と末尾（0.10秒）に自動的に無音パディングを追加。さらに無音と音声の境界部でのプチ音（クリックノイズ）を完全に防止するため、音声の開始・終了時に滑らかな超短時間フェード（5ms = 120サンプル）を適用。
- **CFG（Classifier-Free Guidance）による流暢性と品質向上**:
  - Python リファレンス実装と同等の **CFG（Classifier-Free Guidance）** を C# / ONNX 推論エンジンへ移植実装。各推論ステップで条件付き (Conditional) と無条件 (Unconditional) の2パス実行を行いロジットを補間。GUI に「CFG/ペース」スライダーを追加。
- **参照音声の自動前処理（音量均一化 & トリム）**:
  - 参照音声（ボイスプロンプト）の先頭・末尾の無音区間を自動トリムし、音声振幅のピーク値を目標レベル `-1.0 dB` (≒ `0.89`) に自動スケーリング（音量正規化）。

## 🚀 使い方

### GUI（ウィンドウ起動）
パッケージ内の `CBoxTTS.Native.JA.exe` または `CBoxTTS.Native.EN.exe` をダブルクリックして起動します。
美麗なダークモード対応のモダンUIで、テキスト入力、パラメータ（話速、感情誇張度等）の調整、リアルタイム再生およびWAVファイルへの書き出し・一括保存を行えます。

### 英語専門用語辞書 (`user_dict_en.txt`) のカスタマイズ
ポータブル版の実行ファイルと同じフォルダに配置されている `user_dict_en.txt` をメモ帳などのテキストエディタで編集することで、独自の専門用語や固有名詞の発音を追加・変更できます。

**【辞書のフォーマット（書き方）】**
* 1行につき1単語とし、**`単語,読み方`** の形式（カンマ区切り）で記述します。
* 読み方は英語のアルファベットの自然な発音（例: A → ay, B → bee, C → see）や、そのまま読ませたい英単語で記述します。
* 先頭が `#` で始まる行はコメントとして無視されます。空行も無視されます。
* 大文字・小文字は区別されず、単語境界（前後にスペースや記号がある場合のみ）で正確に置換されます。

**記述例:**
```text
# 一般的な技術用語のスペルアウト
AI,ay eye
IoT,eye oh tee
TTS,tee tee ess

# 独自の固有名詞
MyTech,my tech
```
※ 辞書ファイルを編集した後は、アプリケーションを再起動することで変更が反映されます。

### CLI（コマンドライン起動・テストハーネス）
コマンドプロンプトまたは PowerShell から `--test` 引数付きで実行することで、GUI を起動せずに音声合成の動作確認が可能です。

```powershell
# 日本語版のテスト実行
.\CBoxTTS.Native.JA.exe --test

# 英語版のテスト実行
.\CBoxTTS.Native.EN.exe --test
```
実行すると、カレントディレクトリにテスト結果の音声ファイル（`test_harness_japanese_out.wav` や `test_harness_english_exclusive_out.wav`）が出力されます。

## 🔧 開発・修正履歴（アップデート情報）

本プロジェクトで行われた主要なデバッグおよび修正内容は以下の通りです。

### 1. 話速調整における「声の二重重なり・ダブり」バグの完全解決 (WSOLA 刷新)
- **不具合内容**: 
  - 音声伸縮アルゴリズム（SOLA）において、ピッチ同期のための微調整ズレ（`bestDelay`）が次フレームの基準位置に累積され、長文合成時に再生時間が設定値から大幅にズレる問題がありました。
  - さらに最初のフレームに窓関数（ハニング窓）が適用されていなかったため、後続フレームとの重ね合わせ領域で波形が乱れ、**話速を遅くした際に同じ声が重ね合わさって二重（ディレイ/エコー風）に聞こえる現象**が発生していました。
- **対策**: 
  - 伸縮エンジンをクリーンな **WSOLA（Waveform Similarity Overlap-Add）** アルゴリズムに全面刷新しました。
  - 最初のフレームを含めた全フレームに対して正確にハニング窓を適用し、クロスフェード合成を行うことで声の二重重なり現象を完全に解消。どのような話速設定でも歪みのない自然なテンポで再生されるようになりました。

### 2. 英語発音の訛り（アクセント）解消 & 標準アメリカ英語 (US Native) 化
- **トークン処理バグ**: 英語専用モデルの `Tokenizer.cs` において、文頭の `[START]` (255) トークンの直後に不要な言語トークン `1` (UNK) が自動挿入されていたバグを特定・修正。モデル内部の位置埋め込み（`position_ids`）の1マスズレを解消し、正しくテキストがインデックス0から評価されるように修復しました。
- **参照音声を US Native に更新**: 参照音声（ボイスプロンプト）の同一性問題を解決。`default_voice_en_female.wav` (女性) および `default_voice_en_male.wav` (男性) の標準アメリカ英語プロンプトを新規作成・同梱し、日本語話者アクセントの転写によるカタカナ風の訛りを完全に排除しました。
- **パラメータの自動最適化**: 英語モデル選択時、Exaggeration (抑揚/強調): `0.30`、Temperature: `0.50`、CFG Weight: `0.40` へ自動的に調整されるよう修正しました。

### 3. 日本語・英語 Turbo モデルの Tokenizer 対応 (ByteLevel BPE)
- **改修内容**: Turboモデル等で使用される GPT-2 ベースの ByteLevel BPE トークナイザーに対応するため、[Tokenizer.cs](file:///E:/C-BoxTTS-C/CBoxTTS.Native/Tokenizer.cs) 内にバイトレベル表現（`BytesToUnicode` による UTF-8 バイト ↔ Unicode 文字マッピング）および正規表現事前分割パターンを新たに実装しました。
- **自動検出**: 語彙ファイルの読み込み時に語彙数（`Count > 5000`）を自動検出し、自動で ByteLevel モードを適用します。

### 4. テキスト Embedding 語彙サイズ（`_embedVocabSize`）設定の修正
- **不具合内容**: `embed_tokens.onnx` のEmbeddingテーブルの語彙数制限（`_embedVocabSize`）において、Turboモデル用のサイズが誤って音声トークン辞書サイズと同じ `6563` に制限されていました。これにより、ひらがなや英単語のトークンID（例：`た` -> ID: `25224` 等）が UNK トークン（ID: `1`）にクランプされ無音やノイズが出力されるバグが発生していました。
- **修正内容**: [TTSEngine.cs](file:///E:/C-BoxTTS-C/CBoxTTS.Native/TTSEngine.cs) の `ModelType.Turbo` 用の語彙上限サイズを正しい `50257` へ修正しました。

### 3. ONNX 埋め込み層における境界外アクセス（Gather）クラッシュの解決
- **現象**: 語彙サイズ修正に伴い、ひらがなや句読点の正しいトークンIDが `embed_tokens.onnx` に渡された結果、モデル内部の `/speech_emb/Gather`（語彙サイズ 6563 の音声埋め込みテーブル）で境界外アクセス例外（`idx=16764 must be within [-6563, 6562]`）が発生していました。
- **原因と対策**: ONNX Runtime の Eager Evaluation（先行評価）により、条件分岐で棄却されるはずのブランチにテキストトークンIDが漏れ出ていました。Pythonスクリプトを用いて `embed_tokens.onnx` 内の同ノードの直前に `Clip` ノード（`[0, 6562]` へのクランプ）を挿入するパッチを適用し、安全にクラッシュを解決しました。

### 4. CPU 推論時の並列スレッド制限による高速化（6〜12倍）
- **現象**: GPUが使えない環境やCPUフォールバック実行時、ONNX Runtimeがデフォルトで論理コア数分の大量のスレッドを生成し、スレッド競合とメモリ帯域飽和によって推論（自己回帰ステップ）が極めて低速になる問題がありました。
- **対策**: CPUロードパスにおいて `options.IntraOpNumThreads = Math.Min(4, Environment.ProcessorCount)` を設定し、並列スレッド数を最大4に制限しました。これにより、CPUでのステップ推論速度が **6倍〜12倍近く劇的に向上**（ステップあたり数秒から約250ms〜500msへ短縮）しました。

### 5. cuDNN 探索アルゴリズム変更による CUDA 安定化（クラッシュ対策）
- **現象**: 英語専用モデル等を CUDA 環境で実行した際、初回推論開始の直後に `cudnn64_9.dll` 内部でスタックバッファオーバーラン（例外 `0xc0000409`）が発生し、アプリが強制終了する現象を確認しました。
- **対策**: ONNX Runtime の CUDA プロバイダーオプション内の `cudnn_conv_algo_search` 設定値を、全探索（`EXHAUSTIVE`）から安全かつ高速なヒューリスティック探索（**`HEURISTIC`**）へ変更し、安定動作を確保しました。

### 6. 拡散デコーダーの CPU フォールバック適用（追加の CUDA クラッシュ対策）
- **現象**: 自己回帰ループ（言語モデル推論）が正常終了した直後の波形合成（`conditional_decoder.onnx` の実行）フェーズにおいて、再度 `cudnn_engines_precompiled64_9.dll` のスタックバッファオーバーラン（`0xc0000409`）による強制終了が発生していました。
- **対策**: 拡散デコーダーモデル（`conditional_decoder`）についても `speech_encoder` と同様に CPU で実行するように変更しました。速度に影響の大きい自己回帰ループ（`language_model`）のみを CUDA で実行しつつ、全体のクラッシュを完全に解決しました。

### 7. 話速制御（タイムストレッチ／WSOLA）の安定化と声の二重重なり解消
- **不具合内容**: 
  - 音声伸縮アルゴリズム（SOLA）において、ピッチ同期のための微調整ズレ（`bestDelay`）が次フレームの基準位置に累積され、長文合成時に再生時間が設定値から大幅にズレて「100%（等倍）より90%（ゆっくり）のほうが早く再生が完了する」という速度逆転バグが起きていました。
  - さらに、最初のフレームだけ窓関数（ハニング窓）を掛けずに生データで出力されていたため、後続フレームとの重ね合わせ（クロスフェード）領域で信号波形が正常に合成されず、**話速を遅くした際に同じ声がダブって聞こえる（ディレイやコーラスのような二重重なり）** という音質劣化も発生していました。
- **対策**: 
  - 伸縮エンジンをクリーンな **WSOLA（Waveform Similarity Overlap-Add）** アルゴリズムに全面刷新しました。
  - 最初のフレームを含めた全フレームに対して正確にハニング窓を適用し、クロスフェード合成を行うことで声の二重重なり現象を完全に解消。
  - さらに位相同期探索（相関計算）の対象を出力バッファの正規化済み波形へと修正し、目標出力位置を決定論的に管理することで、どのような話速設定値でも極めてクリアかつ正確なテンポで再生されるようになりました。

### 8. 拡散デコーダー出力の先頭・末尾の無音自動トリム
- **現象**: `conditional_decoder.onnx` の出力長が 117,600 サンプル（4.9秒）で固定化されているため、短い発話の際にテキストの冒頭部分や後半部分に大量の無音領域が生まれ、再生開始時に「発話の頭部分が消えてしまう（頭切れ・無音遅延）」問題が生じていました。
- **対策**: デコーダーから出力された波形に対し、即座に閾値 `0.005f` での無音トリム処理（`TrimSilence`）を実行するように修正しました。実際の発話音声部分のみを的確に抽出し、その後に適切な長さの無音パディングとフェード処理を適用するロジックに統合することで、完璧な冒頭発音を保証しました。

### 9. 反復ペナルティの調整幅拡大
- **機能拡張**: 長文合成時の同じ言葉の繰り返し（自己回帰ループの崩壊）を防ぐため、UIの「反復ペナルティ (Repetition)」スライダーの最大値を `1.5` から **`2.0`** へ拡張しました。長文で繰り返しが起きた際、より強力に抑止可能です（※英語/Turboモデルの初期値も `1.35` に最適化されました）。

### 10. 英語テキスト正規化（EnglishNormalizer）のデバッグと最適化
- **不具合内容・課題**:
  - **スレッド競合とクラッシュ**: `Normalize` 実行時にユーザー辞書の再ロードが他スレッド（音声生成など）と競合し、`InvalidOperationException`（列挙中のコレクション変更）でアプリが突然クラッシュする脆弱性がありました。
  - **動的Regexコンパイルの非効率**: テキスト正規化のたびに 37個の略語に対して `RegexOptions.Compiled` の正規表現オブジェクトをその都度動的生成・コンパイルして破棄しており、CPU・メモリ性能に著しい無駄（アンチパターン）が生じていました。
  - **巨大数値と英語序数のバグ**: `long.MinValue` の絶対値（`long` 上限オーバー）が誤って丸めて読み上げられていた他、1兆以上の単位表現がなく、さらに `101st` などの 100 を超える数の英語の序数（Ordinal）が `one hundred and one th` のように不自然に変換されるバグがありました。
- **対策**:
  - ユーザー辞書のロードとキャッシュ構造をスレッドセーフな設計に改善し、二重チェックロックを導入してスレッド競合によるクラッシュを完全に防止しました。
  - 起動時（静的コンストラクタ）に 37個の略語 Regex を一括プリコンパイルして使い回すように改善し、不要な動的コンパイル負荷を完全に排除してテキスト処理を高速化しました。
  - 内部処理に `ulong` を適用し `long.MinValue` も正確に読み上げられるようにした上で、`trillion`（兆）〜`quintillion`（百京）の単位までサポートを拡張。さらに 100 を超える序数についても `one hundred and first` のように英語圏の正しい慣用読みに沿って動的に変換されるようアルゴリズムを刷新しました。
  - `ms.`（Ms.）の発音マッピングを `miss`（ミス）から `miz`（ミズ）に変更し、コンパイル時に発生していた nullable 警告（CS8600）も完全に解消しました。

### 11. 英語発音の訛り防止とモデル別標準ボイスプロンプトの独立分離
- **不具合内容・課題**: 
  - Zero-shot TTS モデル（Chatterbox）は、ボイスプロンプト参照音声から話者の音韻的アクセント（日本語訛り等）を直接抽出して生成条件として利用します。
  - 従来、全モデルで単一の共通ファイル `models/default_voice.wav` を参照していたため、日本語モデル等でダウンロード・配置された日本語プロンプト音声が英語版でも共通参照され、英語テキスト読み上げ時に**日本語訛り（カタカナ英語風のアクセント）** が強く発生していました。
- **対策**:
  - 英語モデル用ネイティブプロンプト音声（`default_voice_en.wav`）と日本語モデル用プロンプト音声（`default_voice_ja.wav`）を明確に独立分離してダウンロード・配置する構造へ刷新しました。
  - 英語版・英語モデル選択時には自動的に英語ネイティブの参照音声（`default_voice_en.wav`）がセットされ、モデル本来の極めて自然でクリアな標準英語発音が出力されるようになりました。
  - また、日本語話者の参照音声（自分の声など）を保持したまま英語を読ませる場合でも、UIの `CFG Weight` (ペース) を **`0.3` 〜 `0.4`** に調整することで、参照音声のアクセント追従が緩和され、綺麗で聞き取りやすい英語アクセントに補正されます。

### 12. ONNX Runtime 1.20 opset 20 アラインメントと C# 読み込みの完全安定化
- **現象**: ONNX Runtime 1.20 へのライブラリアップデートに伴い、C# の `InferenceSession` 初期化時に `Unsupported model IR version: 10` や `Opset 21 is under development` および `GroupQueryAttention` カスタム演算子の型チェック定義違反（`InvalidGraph`）が発生するケースがありました。
- **対策**:
  - `TTSEngine.cs` の静的コンストラクタで環境変数 `ALLOW_RELEASED_ONNX_OPSET_ONLY = "0"` を自動設定し、未リリース opset ガードを解除。
  - Python スクリプトを用いて全 ONNX モデルの `ir_version` を `9`、インポートするデフォルト opset を `20`（`com.microsoft` opset を `1`）へ整合し、`GroupQueryAttention` ノードの不整合な `softcap` 属性を除去。
  - `language_model.onnx` 内の外部データ保持ファイル名 (`language_model.onnx_data`) をモデル initializer 内部データと同期配置し、CUDA / DirectML / CPU フォールバックいずれの環境でもエラーゼロで高速ロードされるように修復しました。

### 13. 複数行一括WAV保存のファイル命名機能拡張（ゼロ埋め桁揃え連番 & テキスト冒頭付与）
- **機能拡張・改善**:
  - 改行区切りの複数行テキストを一括WAV保存する際、ファイル名で並び替えた時に順番が崩れないよう、総行数に応じたゼロ埋め桁揃え連番（10〜99行なら `_01`, `_02` ... / 100行以上なら `_001` ...）を自動付与するように改良しました。
  - 連番の後ろに各行の読み上げテキスト冒頭（最大10文字、OSのファイル名禁止文字を自動サニタイズ）を付与するロジックを追加。例：`output_01_本日は晴天なり.wav` のようにエクスプローラー上で各ファイルの読み上げ内容を一目で可視化・判別・管理できるようになりました。

### 14. 幻覚生成（ハルシネーション・テキスト外の余計な発声）の根本対策
- **不具合内容・現象**: 数字リスト表記（`1.`, `2)` など）を含む英文テキスト等の読み上げ時に、入力テキストに含まれない意味不明な単語や文章をモデルが自動生成して読み上げてしまう現象（ハルシネーション）が発生していました。
- **対策・改善**:
  - **文結合ロジックの修正**: 数字マーカー（`1.` 等）が単独の音声チャンクとして分離され、文脈欠如によりモデルが暴走していた問題を修復。マーカーを直後の文章と必ず自動結合して合成するように変更しました。
  - **安全フィルタの追加**: 英字を含まない極短チャンク（記号や数値単体）を自動スキップする防御機能を導入。
  - **Position ID の正当化**: 初期テキスト埋め込み時に `position_ids` が負数（`-1`）へ落ち込むバグを修正し、自己回帰推論時の埋め込みズレ（embedding drift）を防止しました。

## 🎛️ UIパラメータ（スライダー）の調整ガイド

アプリケーションの画面上にある各スライダーを調整することで、声質や生成の安定性を細かくコントロールできます。

| パラメータ | 設定範囲 (デフォルト) | 役割と調整の目安 |
| :--- | :---: | :--- |
| **抑揚/表現力** | `0.0` 〜 `1.0` (`0.5`) | 音声の抑揚（ピッチの高低差）の強さ。<br>・**高くする**: 感情豊かで変化のある話し方になります。<br>・**低くする**: 抑揚が抑えられ、ニュース原稿のように平坦で淡々と喋ります。 |
| **話速** | `0.5` 〜 `2.0` (`1.0`) | 話すテンポ（速度）を調整します。（音の高さは維持されます） |
| **温度 (Temperature)** | `0.1` 〜 `1.2` (`0.6`) | 生成音声の「ゆらぎ（ランダム性）」を調整します。<br>・**下げる (`0.2`〜`0.3` おすすめ)**: 推論が決定論的になり、**同一フレーズの繰り返しや吃り、読み飛ばし、ループ現象がほぼ完全に解消**します。<br>・**上げる (`0.8`〜`1.0`)**: 喋り方のゆらぎが人間らしくなりますが、高すぎると吃音やエラーの原因になります。 |
| **CFG/ペース** | `0.0` 〜 `1.0` (`0.5`) | Classifier-Free Guidance の強度。参照音声の特徴（声質）をどの程度忠実に追従するかを決めます。高すぎるとノイジーまたはロボット的になりやすいため、その場合は少し下げます。 |
| **繰り返しペナルティ** | `1.0` 〜 `2.0` (Turbo/EN:`1.35`, 他:`1.2`) | 同じ音声パターンの連続出力を抑制します。音声が途中でループしたり、語尾が「〜〜〜」と伸びてしまう場合は、数値を **`1.3`〜`1.6`** 付近に上げると解消します。 |

## 📦 ビルド方法（ポータブルパッケージの作成）

開発環境（.NET 10 SDK 導入済み）でポータブルパッケージを自らビルド・パブリッシュするには、`CBoxTTS.Native` フォルダ内の以下のスクリプトを PowerShell から実行します。

*   **日本語版のビルド**: `.\build_portable_ja.ps1`
*   **英語版のビルド**: `.\build_portable_en.ps1`

実行すると、自動的に ReadyToRun の最適化ビルドが行われ、必要な依存ライブラリ、モデルファイル、仕様書がそれぞれのポータブルパッケージフォルダへ自動配置されます。
